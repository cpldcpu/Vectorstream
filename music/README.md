# music/ — two-voice TED music engine + tunes

| file | role |
|---|---|
| `player_core.asm` | the engine: `select_tune` / `playtick` (called every 50 Hz tick from the demo IRQ), instrument envelopes, arpeggios, vibrato, slides, master attenuation `musatt` (used by the demo for the latent-pulse fade-out), `write_regs` |
| `player_zp.asm` | the engine's zero-page layout (`$60-$67`), `!src`'d first |
| `song_data.asm` | generated by `songdata.py` — note/instrument tables for all four tunes |
| `songdata.py` | the tunes as Python data, the `song_data.asm` generator, **and** a frame-level cycle-exact reference model of the player |
| `player.asm` | thin standalone wrapper around `player_core.asm` (assembles to `ted_tunes.prg` + `symbols.txt`, both build products) |
| `verify.py` | py65: runs the assembled 6502 player frame by frame against the Python reference model — register-write-exact, all four tunes |

Tunes: 0 **latent pulse** (intro + the whole first part),
2 **night shift** (announcement → finale → held credits),
1 **border glow** and 3 **phosphor pop** (unused spares).

To change a tune: edit `songdata.py`, then `make verify` — the same
source generates both the 6502 data and the model it is verified
against, so the data can't drift from the reference.
